BROWSER FONTS

Font Style

The font-style CSS property allows you to select italic or oblique font-faces within your chosen font family.

Italic forms are generally cursive, while oblique forms are generally sloped.

Values:

  • normal: selects a font that is classified as normal
  • italic: selects an italic form; if that is not available, selects an oblique form
  • oblique: selects an oblique form

Example syntax:

p { font-style: italic }

Font Size

Font-size is the size of font. Font sizes can be measured in pixels, points, em, or percent. The font-size selector will change the size of the font of an element. Changing the font size may change the size of other items since it is used to compute the value of em and ex length units.

Examples of using font size

  • h1 {font-size:50px;}
  • h1 {font-size:2.5em;}

Web-Safe Fonts

These are fonts that work on most computers. If a web-safe font is either your specified font or one of your fallbacks -- fonts the computer displays if the first font specified is not available on the computer -- your webpages will display correctly on the majority of computers.

The complete list of web-safe fonts can be found here: CSS Font Stack.

With mobile devices, however, the list of web-safe fonts is far smaller, at least according to this website:

  • Arial/Helvetica
  • Courier/Courier New
  • Georgia
  • Times/Times New Roman
  • Trebuchet MS
  • Verdana

If you want to use a font that isn't on the list of web-safe fonts, you can use one of the tools we discuss below.

Font Families

The CSS font-family attribute determines which typeface is used. Standard typefaces, such as Courier, Arial, Helvetica, and Georgia can be specified, or generic font styles can be used. Acceptable font styles are serif, sans-serif, monospace, cursive, and fantasy.

Multiple typefaces can be specified for an element and should be separated by commas. If using a font that contains whitespace, the name of that typeface should be in quotes. The font listed first will take priority. If the browser cannot find that font installed on the computer, it will fall to the second option, if the second cannot be fount it will look for the third, etc. If none of the fonts specified can be found the browser's default font family will be used.

It is best practices to end the list of font options with a generic typeface so that you are not relying on the browser's default font. It is also important to be mindful of which fonts are available on which operating systems. For example, Mac computers have the Helvetica font but Windows computers usually do not, and so Arial may be used as a substitute.

Example:
body {
    font-family: Helvetica, Arial, sans-serif;
}

Font Support

When declaring a font family to use on a site, the family of fonts is chosen in a heiarchy. The first choice font will be listed first (it could be an included font), followed by the next choice if that font does not want to render, followed by more choices, then followed by a final choice available to most computers everywhere.

The two most general families of fonts to choose from are serif fonts or sans-serif fonts. Beyond that there are a small variety of web safe fonts to choose from. (Please refer to this page for a full listing.) The designer can then choose their first choice of fonts by using the @font-face rule in most browsers today.
